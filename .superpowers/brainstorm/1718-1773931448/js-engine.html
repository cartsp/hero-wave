JS Engine & Lifecycle

How the canvas animation works and how Blazor manages its lifecycle

JavaScript Module (wavy-background.js)

Exported Functions
// Called from Blazor OnAfterRenderAsync (first render only) export function init(canvasElement, config) → config: { colors, waveCount, waveWidth, blur, speed, opacity, backgroundColor } → Creates simplex noise instance → Sets up canvas context (size, blur filter, composite operation) → Starts requestAnimationFrame loop → Returns instance ID for cleanup // Called when Blazor component is disposed (IAsyncDisposable) export function dispose(instanceId) → Cancels animation frame → Clears canvas → Removes resize listener → Releases references

Animation Loop (internal)

Per-frame rendering
Each frame: 1. Clear canvas → fill with backgroundColor 2. Set globalAlpha to opacity 3. Set filter to blur(Npx) 4. For each wave (i = 0 to waveCount): a. Set strokeStyle to colors[i % colors.length] b. Set lineWidth to waveWidth c. Begin path at x=0 d. For x across canvas width (step ~5px): y = noise(x/800, 0.3*i, time) * 100 + (height * 0.5) e. Stroke the path 5. Increment time by speed factor 6. Request next frame

Blazor Lifecycle

WavyBackground.razor.cs
Implements: IAsyncDisposable Fields: ElementReference _canvas; IJSObjectReference? _module; string? _instanceId; OnAfterRenderAsync(firstRender): if (!firstRender) return; _module = await JS.InvokeAsync<IJSObjectReference>( "import", "./_content/HeroWave/wavy-background.js"); _instanceId = await _module.InvokeAsync<string>( "init", _canvas, BuildConfig()); DisposeAsync(): if (_module != null && _instanceId != null) await _module.InvokeVoidAsync("dispose", _instanceId); if (_module != null) await _module.DisposeAsync();

Simplex Noise

The JS module will include a lightweight simplex noise implementation (~80 lines) inline in the module — no external npm dependencies. This is the standard approach used by the original Aceternity component and keeps the library self-contained.
